Protocolos HTTP, HTTPS y DNS

¿Qué es DNS?

"Domain Name Service", o sistema de nombres de dominio en español, es un sistema que utilizamos para traducir nombres de dominio en IPS y viceversa. Es decir, nos podría servir para asociar la dirección IP 192.332.32.2 con el dominio www.ejemplo.com. Los conceptos que deberemos de tener en cuenta respecto al DNS son: Nombre de dominio, que es el nombre legible para los humanos, Dirección IP, numero con el que se identifica a un equipo o RED, Servidor DNS,que será el que se encarge de guardar en una BBDD las direcciones y sus dominios relacionados, y la resolución DNS, que es el Proceso de traducción de IPs y Nombres.

Funcionamiento

  • 1.- Solicitud.

    Usuario ingresa la dirección.

  • 2.- Consulta.

    El dispositivo envía consulta DNS al servidor DNS.

  • 3.- Búsqueda de respuesta.

    El servidor DNS local consulta si tiene la respuesta en su caché. Sí la tiene, se envía de inmediato. Si no, busca la respuesta en otros servidores. Cuando encuentra la respuesta, guarda la IP en la caché y la envía al usuario.

  • 4.- Respuesta.

    El dispositivo del usuario establece la conexión y visualiza la web.

Aplicaciones del Servicio.

Algunas aplicaciones reales en las que se implementa y utiliza éste servicio pueden ser: Correo electrónico, Autenticación de Correo, Navegación Web o la Implementación de VPN y Redes Privadas.

GIT Y GITHUB.

¿Qué es GIT?

Es un sistema de control de versiones distribuido que funciona localmente y gestiona versiones del código. y puede operar sin conexión a Internet.

¿Qué es GITHUB?

Es una plataforma web que utiliza Git que ofrece servicios de alojamiento para proyectos. Facilita la colaboración, revisión de código y gestión de proyectos y requiere conexión a Internet.

Comandos.

  • Comandos de ayuda.

    1.- git help
    2.- git log (--oneline)
    3.- git status

  • Comandos básicos.

    1.- git innit: Inicializa el staging y repositorio local.
    2.- git add: añadimos archivos al staging.
    3.- git commit: envia los archivos añadidos al repositorio local
    4.- git push: enviamos los cambios al servidor remoto

  • Comandos extra.

    1.- git remote: Comandos relacionados con el repositorio remoto de Github.
    2.- git branch: Crear una rama.
    3.- git checkout: Cambiar de rama.
    4.- git branch -a: Comprobar ramas.

  • Comandos Rollback.

    1.- git clean.
    2.- git reset.
    3.- git clean.
